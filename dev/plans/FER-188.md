# Remove obsolete Factory cutover paths without changing live behavior

## Goal

Remove Factory modules and branches left behind by the lifecycle, planning-handoff, Linear-stage, and review-concurrency cutovers so the code exposes only behavior that production can select. Keep the durable action store as Factory truth and preserve every live contract: item-file and Linear validation text, planning summary content, parallel review execution and step metadata, CLI flags, Factory routes/events/records, publication and Linear projection flows, and hosted Factory behavior. This cleanup must not add a replacement compatibility layer or change live runtime/exported schemas.

## Changes

1. Delete the four unreferenced modules `lib/factory-workspace-changes.ts`, `lib/factory-agent-meta.ts`, `lib/factory-planning-input.ts`, and `lib/factory-lifecycle-writes.ts`. Do not move their APIs or add shims: live workspace mutation checks remain in `lib/review-guard.ts`, planning run agent metadata remains in `lib/factory-planning-run-context.ts`, and lifecycle writes remain owned by `lib/factory-lifecycle-kernel.ts` (with the existing public aliases in `lib/factory-lifecycle.ts`). Typecheck/build discovery is the test seam for stale imports because none of these modules has a live caller.

2. Extract `renderFactoryPlanningSummary` and its private next-action formatting from `lib/factory-planning-handoff.ts` into `lib/factory-planning-summary.ts`, update `lib/factory-planning-run-context.ts` to use that owner, then delete `lib/factory-planning-handoff.ts`. Remove its obsolete meta schema, run loading/mutation, atomic persistence, publication-patch validation, approved-plan artifact validation, and warning merge code rather than recreating them. Current publication and implementation authority stay in `lib/factory-plan-publication.ts` and `lib/factory-implementation-input.ts`; there is no handoff-file compatibility path. Replace `test/factory-planning-handoff.test.ts` with focused exact-output coverage in `test/factory-planning-summary.test.ts` for the live summary variants (handoff fields and next action, reviewed iterations, dry-run/empty state, human questions, and error) so extraction cannot change summary bytes or content.

3. Simplify `lib/factory-triage-input.ts:resolveFactoryWorkItemInput` to validate the same mutually exclusive source choice and then either read the item file or fetch the Linear work item directly. Remove `FactoryLifecycleReadMode`, lifecycle roots, workspace-local escape hatches, lock options, `mergeLifecycleState`, lifecycle warnings, and unused triage-named aliases; remove the now-unused `lib/factory-lifecycle.ts:FactoryLifecycleWarning` type and keep the existing item-file, Linear configuration, and `LINEAR_API_KEY` errors exactly unchanged. Update `bin/factory-commands.ts`, `bin/factory-planning-cli.ts`, and `bin/factory-implementation-cli.ts` to stop passing the removed no-op inputs. In `bin/factory-commands.ts:fetchFactoryLinearWorkItem` and `bin/factory-triage-cli.ts`, remove the synthetic state-root override and unreachable warning envelope while returning the fetched work item unchanged. Keep `factory linear fetch`'s registered `--factory-store-root` and `--factory-store-project-id` flags and current validation order: continue resolving those overrides only to preserve their existing validation, but do not pass a derived root into work-item resolution or merge lifecycle state. Retain `lib/factory-run-context.ts:stripFetchWarnings` as the existing historical item-file compatibility behavior, updating only its comment if it otherwise claims current fetches emit warnings. Adjust `test/factory-linear-fetch.test.ts` to cover the live direct-fetch/no-write contract, and rely on the existing `test/cli.test.ts` and `test/factory-implementation-cli.test.ts` cases for unchanged source-selection, item-file, Linear config, and API-key errors.

4. Delete `lib/factory-linear-planning-apply.ts:linearPlanningAttentionStageFromComments` and its now-private comment sorting/date helpers, then remove only the parser import and isolated parser test from `test/factory-linear-adapter.test.ts`. Keep the existing adapter tests that prove fetched/listed Linear items do not acquire `factoryStage` from comments; Linear comments remain projection text and marker-based idempotency inputs, never Factory state.

5. In `workflows/review-steps.ts`, always start the selected review tasks together and await all settled results. Remove the optional serial selector from the workflow-facing `WorkflowContext`, the serial runner, and serial-only executed-step trimming. Keep `lib/workflow-context.ts`'s existing returned `reviewConcurrency` field as the literal `"parallel"` (removing only the misleading resolver whose type admits `"serial"`) because workflow-context return fields are out of scope. In `test/review-steps.test.ts`, remove the synthetic serial setup and two serial-only tests while retaining the live assertions that all requested reviewers start before either resolves, failures wait for all selected reviewers, and requested/executed/omitted metadata stays unchanged. Keep one `test/workflow-context.test.ts` assertion that the production context reports `parallel` and remove only redundant concurrency coverage.

6. In `bin/harness.ts:addPlanReviewCommand` and `addReviewCommand`, replace the two duplicated Codex-only flag checks with `bin/cli-validation.ts:assertCodexOnlyAgentOptions` at the same point in each command's validation order. Pass the resolved provider plus the user-supplied executable, sandbox, approval policy, and reasoning effort so both existing error strings remain byte-for-byte unchanged and Codex/config-only values retain current behavior. Extend the CLI integration cases in `test/cli.test.ts` to exercise both plan-review and change-review call sites for policy flags and executable overrides; do not add a lower-level duplicate test for the helper.

## Verify

- Run `pnpm exec vitest run test/factory-planning-summary.test.ts test/factory-linear-fetch.test.ts test/factory-linear-adapter.test.ts test/review-steps.test.ts test/workflow-context.test.ts test/cli.test.ts test/factory-implementation-cli.test.ts` for the affected live contracts.
- Run `make check` for formatting, lint, typecheck, build, and the normal test gate.
- Run `make smoke-factory` because Factory lifecycle, input, planning-summary, and review workflow code changed.
- Run `git diff --check` as the acceptance whitespace check.

## Boundaries

- Do not change Factory routes, lifecycle events, action-store records, publication/projection behavior, hosted Inngest/Grove behavior, or any remaining persisted/runtime/exported schema outside the deleted handoff loader.
- Do not remove accepted CLI flags, `human_required`, V1 readers, review summary fields, agent raw provenance, successful-result usage, or workflow-context return fields; the fixed `reviewConcurrency: "parallel"` return field remains even though review execution no longer branches on it.
- Do not turn this into broader compatibility cleanup: retain historical item-file warning stripping and all current schema readers, and do not add a generalized cleanup framework or unrelated refactor.
